# Public Review Checklist

This checklist is generic and uses no live project information. It is designed to invite feedback on review quality, missing fields, and validation rules.

## Evidence

- Source material availability
- Source material date
- Source material owner type
- Independent confirmation status
- Version consistency
- Missing attachment list

## Assumptions

- Assumption source
- Assumption owner
- Reason for using the assumption
- Review date
- Sensitivity to change
- Replacement evidence required

## Risks

- Missing source material
- Unclear approval path
- Unclear dependency
- Third-party dependency
- Timeline uncertainty
- Responsibility gap
- Version mismatch

## Controls

- Required fields
- Review owner
- Evidence owner
- Change log
- Approval note
- Exception note
- Review expiry date

## Feedback Request

Which review checks are missing, duplicated, or too vague? Which ones should be mandatory before a project moves to the next review stage?
