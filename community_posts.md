# Community Posts

Use these drafts to attract useful feedback without sharing protected material.

## GitHub Discussion Draft

```text
We are building a public, fake-data project review workflow.

Goal:
Improve the structure for separating FACT, ASSUMPTION, RISK, MISSING ITEMS, and CONDITIONS before a project moves to the next review stage.

What we are sharing:
- fake sample JSON
- minimal public input schema
- generic review checklist

What we are not sharing:
- live project records
- private organization information
- client names
- private status updates
- protected documents
- formulas or decision methods
- production instructions
- access material

Please comment on:
1. Missing fields in the schema
2. Weak validation rules
3. Risk categories that should be split or added
4. Edge cases an assisted review workflow should handle
5. Output format improvements for review teams

Best comments are specific and actionable.
```
